# 기술적으로 가장 어려웠던 문제와 극복 과정

## 문제 상황
콘서트 좌석 예약 시스템에서 동시에 100명이 같은 좌석을 예약할 때 오직 1명만 성공해야 하는 동시성 제어가 가장 큰 과제였습니다. 초기에는 Adapter 레벨에서 수동으로 분산락을 관리했으나, 트랜잭션이 먼저 시작된 후 락을 획득하는 잘못된 순서로 인해 DB 커넥션이 락 대기 시간(최대 9초) 동안 불필요하게 점유되어 커넥션 풀 고갈 위험이 있었습니다. 또한 120줄의 복잡한 락 관리 코드와 데드락 위험도 존재했습니다.

## 해결 과정
AOP 기반 분산락 시스템으로 전면 리팩토링했습니다. @Order 설정으로 분산락 획득을 트랜잭션보다 먼저 실행하도록 순서를 보장하고, SpEL 표현식으로 좌석별 개별 락 키를 동적 생성해 병렬 처리를 유지했습니다. 데드락 방지를 위해 AOP에서 락 키를 자동 정렬하여 모든 스레드가 동일한 순서로 락을 획득하도록 했습니다.

## 결과
DB 커넥션 점유 시간이 4000ms에서 1000ms로 75% 개선되었고, 코드는 120줄에서 48줄로 60% 감소했습니다. 통합 테스트에서 100명이 동시 예약 시 1명만 성공하는 것을 검증했으며, 서로 다른 좌석은 병렬 처리되어 처리량도 유지했습니다. 관심사 분리를 통해 유지보수성도 크게 향상되었습니다.
